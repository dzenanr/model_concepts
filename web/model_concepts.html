Overview

With the Model Concepts tool you may create a model of concepts with attributes and relationships. Click the concept tool in the tool bar (brown background) to create a concept by clicking on an empty space in the model. To move a concept, click the concept and keep the mouse down while moving it. Click the concept to select or deselect it.

The selected concept is displayed in the tool bar. A name of the selected concept may be entered in the Concept field of the tool bar. Use the Enter key to see the name in the concept's title. A concept attribute may be created by entering its name in the attribute field of the tool bar. An existing attribute may be selected by entering its name. The next attribute is obtained by clicking on the next button.

Use the relationship tool to create a relationship between the last two selected concepts by clicking on an empty space in the model. Click the relationship to select or deselect it. The selected relationship is displayed in the tool bar.

Name the current model in the Model menu and save it in a local storage of your browser. Later on, enter the model name and open it.

Objective

The Model Concepts tool is developed in Dart to start learning Dart. The tool is based on Modelibra Modeler developed in Java.

The tool is developed in a step-by-step approach, spiral by spiral.

Spirals

The spiral approach to software learning and development, which preserves a project history as a series of code snapshots or spirals (under the former name of Magic Boxes), is used in this project.

Learning new software concepts and technologies is a challenging task. Learning in spirals, from simple to more advanced concepts but with concrete software applications, helps students get a reasonable confidence level early on, and motivates them to learn by providing more useful applications. With each new spiral, the project grows and new concepts are introduced. A new spiral is explained with respect to the previous one. The difference between two consecutive spirals is that the next spiral has the new code introduced and the old code modified or deleted. This is called learning by anchoring to what we already understand. With a new spiral, we can come back to what we did previously and improve it. In this way, learning in spirals can touch the same topic several times, but each time with more details in a better version.

There are many books where students have to learn quite a lot before applying new concepts, and even then, it is not obvious how to develop a complete software application. It took me more than ten years of learning and teaching to find out that the initial learning of a new technology must be task driven and not topic (subject) driven. Most software books are topic driven. It takes a quick look at the Table of Contents of almost any software book to realize that each chapter introduces a major topic. In a spiral, there may be moreselected than one topic and all of them relate to what we want to accomplish with the spiral.

Tiny User Guide

Click the concept tool to create a concept by clicking on an empty space in the model. A concept must be selected to enter its name in the Concept field of the tool bar. Use the Enter key to see the name in the concept's title.

To move a concept, click the concept and keep the mouse down while moving it. The connected relationships will follow.

Click the unselected concept to select it. Click again to deselect it. The selected concept is displayed in the tool bar. If there are several selected concepts (and/or relationships), you may deselect them all by clicking on an empty space in the model. For the last selected concept, its title (name) may be changed by entering text in the Concept field of the tool bar. The concept may be declared as entry into the model.

A new attribute may be added to the last selected concept by entering its name in the attribute field. Use the Enter key to see the name in the concept's attributes. If an attribute name is oid or code, it will not be generated as a specific attribute in EDNetCore, because every concept inherits both the oid (object id generated automatically by ednet_core) and code (an id that may be null, but if non-null it must be unique) properties. An attribute may be declared as guid (globally unique identifier--not significant to a user), identifier (significant to a user) or required (mandatory value). The choice is presented in the pop-up list. A default type for an attribute is String. An attribute type may be changed by selecting a different type from the pop-up list. An attribute may have an initial (default) value. In the attribute init field, you may use increment for incrementing a value of the int type in EDNetCore by increment starting with increment. The empty text indicates that a default value will be a non-null empty String. For the DateTime type, the now value determines the today's current time at the moment of adding an entity in EDNetCore. An essential attribute is one of the most important attributes of the concept. In a default application, essential attributes will be displayed in a list (or a table) of attributes. A value of a sensitive attribute will be displayed as asterisks. An existing attribute may be selected, to become the current attribute, by entering its name, which is then visually selected. The next attribute is obtained by the next button. The previous attribute is obtained by the previous button. The current attribute may be changed, including its sequence position within the concept (move down or move up buttons), or even removed (remove button). The size of selected concepts may be changed by menu items in the View menu.

If you want to create several concepts, double-click the concept tool to stay on. To return to the select mode double-click the select tool.

Use the relationship tool to create a relationship between the last two selected concepts by clicking on an empty space in the model. The last selected concept is a child and before last is a parent. By default, the parent concept has 0..N cardinalities. The min is 0 and the max is N. By default, the child concept has 1..1 cardinalities. An example is Department--0..N----1..1--Employee. A department has from 0 to N employees and an employee works for exactly one department.

A relationship has two directions: from parent to child (neighbor) and from child to parent (neighbor). You cannot create more than two relationships between the same two concepts. The selected relationship may be transformed (but only if involved concepts are named) to inheritance relationship (child inherits from parent), to reflexive relationship (one concept) or to twin relationships (two relationships between the same two concepts must be first created and the last one selected before they are separated as twins). The choice is presented in the pop-up list. By default, a new relationship is internal. The child concept of the internal relationship is not an entry point into the model. The relationship cardinalities may be changed. For the 1..1 cardinalities, the identifier (id) may be checked on.

Click the unselected relationship, or very close to it, to select it. Click again to deselect it. The selected relationship is displayed in the tool bar. A reflexive relationship is selected by clicking on a horizontal portion of the relationship. If two relationships are created between the same two concepts, the last relationship may be selected by clicking on both relationships. The twin choice will separate the two relationships into twins. If there are several selected relationships (and/or concepts), you may deselect them all by clicking on an empty space in the model.

If you want to create several relationships, double-click the relationship tool to stay on. To return to the select mode double-click the select tool.

Name the current model in the Model menu and save it in a local storage of your browser. Later on, enter the model name and open it.

Transform a model, but only non-hidden concepts and relationships, to json by clicking on the from model to json button above the json panel. Copy the json text from the json panel into a local text file to save the model. Later, paste the json text into the json panel and click on the from json to model button to obtain the model.

For a pretty json version of the model, click on the pretty json button, but only after clicking on the from model to json button.

Transform a model to an image by clicking on the to image button in front of the image panel. Copy the image to a local text file.
